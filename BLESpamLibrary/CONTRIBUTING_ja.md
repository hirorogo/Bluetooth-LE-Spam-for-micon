# BLESpamライブラリへの貢献

> [日本語版](CONTRIBUTING_ja.md) | [English](CONTRIBUTING.md)

BLESpamライブラリへの貢献に興味を持っていただきありがとうございます！このドキュメントは貢献のためのガイドラインを提供します。

## 貢献方法

### バグ報告

バグを見つけた場合は、GitHubでissueを作成してください。以下を含めてください:
- バグの明確な説明
- 再現手順
- 期待される動作
- 実際の動作
- ハードウェア（ボードモデル）
- ソフトウェアバージョン（Arduino IDE、ESP32ボードパッケージ）
- 問題を示すコード

### 機能提案

機能リクエストを歓迎します！以下を含むissueを作成してください:
- 機能の明確な説明
- ユースケース / なぜ必要か
- 提案する実装（アイデアがあれば）

### コード貢献

プルリクエストを歓迎します！以下のガイドラインに従ってください:

1. **リポジトリをフォーク**
   ```bash
   git clone https://github.com/YOUR_USERNAME/Bluetooth-LE-Spam-for-micon.git
   cd Bluetooth-LE-Spam-for-micon
   ```

2. **機能ブランチを作成**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **変更を加える**
   - 既存のコードスタイルに従う
   - 複雑なロジックにコメントを追加
   - 必要に応じてドキュメントを更新

4. **変更をテスト**
   - ターゲットプラットフォームでコンパイルを確認
   - 可能であれば実機でテスト
   - 検証スクリプトを実行:
     ```bash
     cd BLESpamLibrary
     bash extras/validate_structure.sh
     ```

5. **変更をコミット**
   ```bash
   git add .
   git commit -m "機能追加: 変更の説明"
   ```

6. **プッシュしてプルリクエストを作成**
   ```bash
   git push origin feature/your-feature-name
   ```
   その後、GitHubでプルリクエストを作成

## コードスタイルガイドライン

### C++コード
- インデントに4スペースを使用（タブなし）
- 開き括弧は同じ行に
- 明確な変数名
- 複雑なアルゴリズムにコメント

例:
```cpp
void BLESpam::someFunction(uint8_t parameter) {
    // これが何をするかを説明するコメント
    if (parameter > 0) {
        doSomething();
    }
}
```

### Arduinoスケッチ
- 例はシンプルでよくコメントされたものにする
- 先頭に説明を含める
- デバッグにはシリアル出力を使用
- 可能な限り1つの例につき1つの機能を示す

### ドキュメント
- 明確で簡潔な言語を使用
- コード例を含める
- READMEファイルを最新に保つ
- CHANGELOG.mdにエントリを追加

## 貢献の分野

以下の分野での貢献を特に歓迎します:

### 1. プラットフォームサポート
- nRF52シリーズの実装
- nRF Connect SDKサポート
- Zephyr RTOSサポート
- その他のBLE対応プラットフォーム

### 2. 機能
- 追加のデバイスタイプとスパムパターン
- Fast Pair拡張サポート
- Lovespouseデバイスサポート
- BLEスキャナー/検出機能
- Webベースの設定インターフェース
- 電力管理の最適化

### 3. 例
- 高度な使用例
- プラットフォーム固有の例
- 実世界のユースケースのデモンストレーション

### 4. ドキュメント
- ビデオチュートリアル
- 配線図
- トラブルシューティングガイド
- 他言語への翻訳

### 5. テスト
- 単体テスト
- ハードウェア互換性テスト
- 通信距離とパフォーマンスのベンチマーク

## 開発環境のセットアップ

### 必要なソフトウェア
- Arduino IDE 1.8.13以降またはArduino IDE 2.x
- ESP32ボードサポートパッケージ
- バージョン管理用のGit

### 推奨ツール
- PlatformIO（高度な開発用）
- Visual Studio Code
- シリアルモニターツール

### テストハードウェア
- ESP32開発ボード
- テスト用のターゲットデバイス（オプションだが推奨）

## プルリクエストプロセス

1. 変更の詳細でCHANGELOG.mdを更新
2. 必要に応じてドキュメント（README、例）を更新
3. すべての例が正常にコンパイルされることを確認
4. 可能であれば実機でテスト
5. メンテナーにレビューをリクエスト

## 行動規範

### 私たちの基準
- 敬意を払い、包括的である
- 建設的な批判を受け入れる
- コミュニティにとって最善のことに焦点を当てる

### 受け入れられない行動
